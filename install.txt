4. 실행 명령어
----------------------------------------------------------------

  [ 프론트엔드 ]

  # 1) 디렉토리 이동
  cd frontend

  # 2) 패키지 설치
  npm install

  # 3) 개발 서버 시작 (http://localhost:3000)
  npm run dev

  # 4) 프로덕션 빌드
  npm run build

  # 5) 빌드 결과 미리보기
  npm run preview


  [ 백엔드 ] (선택사항 — 현재는 목 데이터로 동작)

  # 1) 디렉토리 이동
  cd backend

  # 2) 패키지 설치
  npm install

  # 3) 개발 서버 시작 (http://localhost:4000, 파일 변경 감지)
  npm run dev

  # 4) 프로덕션 실행
  npm start


----------------------------------------------------------------
5. 처음부터 실행하는 전체 순서
----------------------------------------------------------------

  git clone <repository-url>   # 또는 폴더 복사
  cd Antigravity

  # 프론트엔드
  cd frontend
  npm install
  npm run dev
  # → 브라우저에서 http://localhost:3000 접속

  # (선택) 백엔드 — 별도 터미널에서
  cd ../backend
  npm install
  cp .env.example .env         # .env 파일 생성 후 Supabase 키 입력
  npm run dev


----------------------------------------------------------------
6. 환경변수 (.env)
----------------------------------------------------------------

  [ frontend/.env ]
  VITE_SUPABASE_URL=https://your-project-ref.supabase.co
  VITE_SUPABASE_ANON_KEY=your-anon-key-here
  VITE_API_BASE_URL=http://localhost:4000

  ※ .env 파일이 없어도 목 데이터(src/data/mockData.js)로
     모든 페이지가 정상 동작합니다.


----------------------------------------------------------------
7. 데이터베이스 스키마 적용 (Supabase 연결 시)
----------------------------------------------------------------

  supabase/migrations/001_initial_schema.sql 파일을
  Supabase Dashboard → SQL Editor 에서 실행

  포함 테이블:
    partners    파트너 기본 정보
    contracts   계약 정보 (히스토리)
    revenues    매출 데이터 (시계열)
    activities  활동 로그 (CRM)
    risk_scores 리스크 점수

  포함 뷰(View):
    vw_monthly_revenue  월별 매출 집계
    vw_partner_ltv      파트너별 LTV
    vw_retention        계약 유지율
